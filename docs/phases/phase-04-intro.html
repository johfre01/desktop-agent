Drag & Resize Functionality

Making the overlay interactive with mouse events

App Objective

Enable users to drag the frame anywhere on screen and resize it from any edge or corner.

Learning Objective

Master mouse event handling, window geometry manipulation, hit testing, and cursor management.

The Challenge

Since we removed the default window frame in Phase 3, we lost the built-in ability to drag and resize our window. Now we need to implement these features ourselves using mouse events.

This involves three main tasks:

  • Hit Testing: Determining which part of the window the user clicked
  • Dragging: Moving the window when the user drags from the center
  • Resizing: Changing window size when the user drags from edges/corners

Mouse Events in Qt

Qt provides three main mouse events that we'll override:

mousePressEvent
Button pressed down
→
mouseMoveEvent
Mouse moves (while pressed)
→
mouseReleaseEvent
Button released

Event Methods

def mousePressEvent(self, event):
    """Called when mouse button is pressed."""
    if event.button() == Qt.MouseButton.LeftButton:
        # Store starting position
        self.drag_start = event.globalPosition().toPoint()

def mouseMoveEvent(self, event):
    """Called when mouse moves (with button pressed)."""
    if event.buttons() & Qt.MouseButton.LeftButton:
        # Calculate how far the mouse moved
        delta = event.globalPosition().toPoint() - self.drag_start
        # Move the window by that amount
        self.move(self.pos() + delta)
        self.drag_start = event.globalPosition().toPoint()

def mouseReleaseEvent(self, event):
    """Called when mouse button is released."""
    self.drag_start = None
Global vs Local Position: globalPosition() gives coordinates relative to the entire screen. position() gives coordinates relative to the widget. For dragging windows, we need global coordinates.

Hit Testing: Where Did They Click?

We need to determine which zone the user clicked in to decide whether to drag or resize (and which direction to resize).

NW N NE W DRAG E SW S SE

9 zones: 4 corners + 4 edges + center (drag area)

The Edge Margin

We define a "margin" (e.g., 8 pixels) at the edges. Clicks within this margin trigger resizing; clicks in the center trigger dragging. 

EDGE_MARGIN = 8  # Pixels from edge for resize zones

def get_edge_at_position(self, pos):
    """Determine which edge/corner the position is in."""
    x, y = pos.x(), pos.y()
    width, height = self.width(), self.height()

    left = x < EDGE_MARGIN
    right = x > width - EDGE_MARGIN
    top = y < EDGE_MARGIN
    bottom = y > height - EDGE_MARGIN

    if top and left: return "top-left"
    if top and right: return "top-right"
    # ... etc

Cursor Feedback

To give users visual feedback about what will happen when they click, we change the cursor based on which zone they're hovering over.

Zone Cursor Qt Constant
Center (drag) Move arrows (✥) Qt.CursorShape.SizeAllCursor
Left / Right edges Horizontal arrows (↔) Qt.CursorShape.SizeHorCursor
Top / Bottom edges Vertical arrows (↕) Qt.CursorShape.SizeVerCursor
Top-left / Bottom-right Diagonal (↘) Qt.CursorShape.SizeFDiagCursor
Top-right / Bottom-left Diagonal (↙) Qt.CursorShape.SizeBDiagCursor 
# Change cursor based on position
self.setCursor(Qt.CursorShape.SizeHorCursor)
User Experience: Changing cursors is crucial for usability. Without visual feedback, users won't know they can resize from the edges.

Resizing Logic

Resizing is more complex than dragging because we need to adjust both position and size, and different edges affect different properties:

  • Right edge: Only width changes
  • Bottom edge: Only height changes
  • Left edge: X position AND width change
  • Top edge: Y position AND height change
# Resizing from the right edge
new_width = self.width() + delta.x()
self.resize(new_width, self.height())

# Resizing from the left edge (position also changes!)
new_x = self.x() + delta.x()
new_width = self.width() - delta.x()
self.setGeometry(new_x, self.y(), new_width, self.height())
Minimum Size: Always respect the minimum size when resizing! Check that the new dimensions don't go below minimumWidth() and minimumHeight().

Mouse Tracking

By default, mouseMoveEvent is only called when a mouse button is pressed. To update the cursor while hovering (no button pressed), we enable mouse tracking: 

# In __init__ or init_ui
self.setMouseTracking(True)

With this enabled, mouseMoveEvent fires continuously as the mouse moves, allowing us to update the cursor based on position.

The Complete Flow

  1. Mouse moves → Update cursor based on zone (with mouse tracking)
  2. Mouse pressed → Store start position and determine action (drag or resize + direction)
  3. Mouse moves (button held) → Execute drag or resize based on stored action
  4. Mouse released → Clear stored state

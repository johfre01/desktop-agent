Qt Style Sheets (QSS)

QSS is Qt's styling system, similar to CSS for web. Key features we used:

Gradient Backgrounds

background: qlineargradient(x1:0, y1:0, x2:1, y2:0,
    stop:0 #00d9ff, stop:1 #00ff88);

Pseudo-States

QPushButton:hover { /* When mouse hovers */ }
QPushButton:pressed { /* When button is clicked */ }
QLineEdit:focus { /* When input has focus */ }
QLineEdit::placeholder { /* Placeholder text styling */ }

QSS vs CSS

QSS is similar to CSS but not identical. Key differences: use qlineargradient() instead of linear-gradient(), pseudo-elements use ::, and not all CSS properties are supported. When in doubt, check the Qt documentation.

Signals and Slots

Qt's signal-slot mechanism connects events to handlers:

# Connect button click to our method
self.capture_button.clicked.connect(self.on_capture_clicked)

# Connect Enter key in input to our method
self.chat_input.returnPressed.connect(self.on_chat_submit)

The Pattern

widget.signal.connect(handler_method) - When the signal fires, the handler is called. No need to manually check for events or poll for changes.

Widget Composition

We created ControlPanel as a separate class, then added it to MainWindow. This pattern keeps code organized:

class ControlPanel(QWidget):
    """Self-contained panel with its own layout and widgets."""
    def __init__(self, parent=None):
        super().__init__(parent)
        # Set up button, input, layout...

class MainWindow(QWidget):
    def init_ui(self):
        # Create and add the panel
        self.control_panel = ControlPanel(self)
        main_layout.addWidget(self.control_panel)

Why Composition?

Each widget class handles its own setup and styling. The parent just places it in a layout. This makes code easier to understand, test, and modify.

Bonus Feature: Screen Boundary Constraints

User-Requested Enhancement

During testing, we added logic to prevent dragging the window off-screen. The window now stops at screen edges.

# Get screen boundaries (respects taskbar)
screen = QApplication.primaryScreen().availableGeometry()

# Constrain position to keep window on screen
new_x = max(screen.left(), min(new_pos.x(), screen.right() - self.width()))
new_y = max(screen.top(), min(new_pos.y(), screen.bottom() - self.height()))
"Good UI is invisible - users focus on their task, not fighting the interface."

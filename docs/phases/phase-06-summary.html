The Flicker-Free Technique

The naive approach (hide window, capture, show window) causes visible flicker. Our solution:

Transparency Toggle

Instead of hiding the entire window, we temporarily skip drawing the nearly-invisible interior fill. The window stays visible (border + panel), but the interior becomes fully transparent, allowing the screen capture to see through.

# In paintEvent - skip fill during capture
if not self._is_capturing:
    painter.fillRect(frame_rect, QColor(0, 0, 0, 1))

# Capture flow
self._is_capturing = True
self.repaint()                    # Make interior transparent
QApplication.processEvents()       # Ensure repaint completes
QTimer.singleShot(50, self._do_capture)  # Brief delay for compositor

The Pixel-Perfect Challenge

Why Was This Tricky?

The border is drawn with QPainter.drawLine() using a pen width of 2. The pen is centered on the line coordinates, extending 1 pixel on each side. Combined with asymmetric border adjustments and anti-aliasing, calculating the exact capture region required careful analysis.

Border Pixel Coverage

For BORDER_WIDTH = 2 with pen centered on offset 1:

Edge Line Position Pixels Covered Margin Needed
Left x = 1 0, 1, 2 3 (BORDER_WIDTH + 1)
Top y = 1 0, 1, 2 3 (BORDER_WIDTH + 1)
Right x = width - 3 w-4, w-3, w-2 5 (BORDER_WIDTH + 3)*
Bottom y = height - 2 h-3, h-2, h-1 4 (BORDER_WIDTH + 2)*

* Extra margin for asymmetric border adjustment and anti-aliasing

Final Margin Formula

left_margin   = BORDER_WIDTH + 1   # 3 pixels
top_margin    = BORDER_WIDTH + 1   # 3 pixels
right_margin  = BORDER_WIDTH + 3   # 5 pixels
bottom_margin = BORDER_WIDTH + 2   # 4 pixels

Key Takeaways

1. QTimer.singleShot() for Async Operations

Never use time.sleep() in a Qt application - it freezes the event loop. Use QTimer.singleShot(delay_ms, callback) to schedule operations after a delay while keeping the UI responsive.

2. Screen Capture with QScreen

QApplication.primaryScreen().grabWindow(0, x, y, w, h) captures a region of the desktop. The first parameter 0 means "entire desktop" rather than a specific window ID.

3. Pen Width Extends Both Directions

A line drawn at coordinate C with pen width W covers pixels from C - W/2 to C + W/2. This is critical when calculating exact pixel boundaries.

4. Anti-Aliasing Bleeds Color

With anti-aliasing enabled, colored pixels can "bleed" into adjacent pixels as semi-transparent values. Always add a buffer pixel when precision matters.

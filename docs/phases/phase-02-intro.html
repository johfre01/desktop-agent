What is PyQt6?

PyQt6 is a set of Python bindings for the Qt framework, one of the most powerful and widely-used GUI toolkits in the world. Qt is written in C++ and powers applications like VLC, VirtualBox, and even parts of the KDE desktop environment.

PyQt6 lets us use all of Qt's power with Python's simplicity. We can create windows, buttons, text inputs, and complex custom widgets - everything we need for Desktop Agent.

Why PyQt6?

Cross-Platform

Feature-Rich

Supports transparency, custom painting, system tray, and more.

The Core Components

QApplication - The Application Manager

Every PyQt application needs exactly one QApplication instance. It manages:

  • Application-wide settings and resources
  • The event loop (processing user input, repaints, etc.)
  • Command-line argument handling
  • Application lifecycle (startup and shutdown)
from PyQt6.QtWidgets import QApplication
import sys

# Create the application - must happen before any widgets
app = QApplication(sys.argv)

# ... create windows and widgets here ...

# Start the event loop - this blocks until the app exits
sys.exit(app.exec())
Key Point: app.exec() starts the event loop and doesn't return until the application quits. Everything after it only runs when the app closes.

QWidget - The Building Block

QWidget is the base class for all UI elements in Qt. A widget can be:

  • A window (if it has no parent)
  • A child widget inside another widget (buttons, labels, etc.)
  • A custom component you design yourself
from PyQt6.QtWidgets import QWidget

class MainWindow(QWidget):
    def __init__(self):
        super().__init__()  # Initialize the parent class

        # Set window properties
        self.setWindowTitle("Desktop Agent")
        self.setGeometry(100, 100, 800, 600)  # x, y, width, height

The Event Loop

GUI applications work differently from simple scripts. Instead of running top-to-bottom and exiting, they wait for events (mouse clicks, key presses, window resizes) and respond to them.

(click, type, etc.)

The event loop runs continuously, checking for events and dispatching them to the appropriate widgets. When you call app.exec(), you're starting this loop.

Think of it like: A receptionist who continuously checks for incoming calls and routes them to the right person. The receptionist never stops working until the office closes.

Signals and Slots

Qt uses a pattern called "signals and slots" for communication between objects. It's one of Qt's most powerful features and essential for building interactive applications.

  • Signal: An announcement that something happened (e.g., "button was clicked")
  • Slot: A function that responds to a signal (e.g., "take a screenshot")
# Connecting a signal to a slot
button.clicked.connect(self.take_screenshot)

# When button is clicked, take_screenshot() is automatically called
def take_screenshot(self):
    print("Screenshot captured!")

Why Signals and Slots?

Traditional Callbacks Signals and Slots
Tight coupling between objects Loose coupling - objects don't need to know about each other
One-to-one connections One signal can connect to many slots
Manual cleanup needed Automatic cleanup when objects are deleted
We'll use this later: In Phase 5, we'll connect our Capture button's clicked signal to a screenshot function. For now, just understand the concept.

Window Geometry

Every widget has a position and size, collectively called its "geometry". For top-level windows, this is relative to the screen. 

# Method 1: setGeometry(x, y, width, height)
window.setGeometry(100, 100, 800, 600)

# Method 2: Separate calls
window.move(100, 100)      # Position
window.resize(800, 600)    # Size

# Method 3: Fixed size (can't be resized by user)
window.setFixedSize(800, 600)

Showing and Closing Windows

Showing a Window

Creating a widget doesn't automatically display it. You must call show(): 

window = MainWindow()
window.show()  # Now the window appears on screen

Closing a Window

Windows can be closed by:

  • Clicking the X button (generates a close event)
  • Calling window.close() programmatically
  • Pressing Alt+F4 (Windows) or Cmd+Q (macOS)
Default Behavior: When the last window is closed, the application automatically quits and app.exec() returns.

Putting It All Together

Here's the complete pattern for a basic PyQt6 application: 

import sys
from PyQt6.QtWidgets import QApplication, QWidget


class MainWindow(QWidget):
    """Main application window."""

    def __init__(self):
        super().__init__()
        self.init_ui()

    def init_ui(self):
        """Initialize the user interface."""
        self.setWindowTitle("Desktop Agent")
        self.setGeometry(100, 100, 800, 600)


def main():
    # 1. Create the application
    app = QApplication(sys.argv)

    # 2. Create and show the main window
    window = MainWindow()
    window.show()

    # 3. Start the event loop
    sys.exit(app.exec())


if __name__ == "__main__":
    main()

Key Takeaways

1. Virtual Environments Isolate Dependencies

Each project gets its own "sandbox" with specific package versions. This prevents conflicts between projects and makes your code reproducible on other machines.

2. Project Structure Matters

The src/ layout is a modern best practice. It keeps your package code separate from configuration files and prevents import confusion during development.

3. Git Tracks History, GitHub Hosts It

Git is the version control tool that runs locally. GitHub is a cloud service that hosts your repository, enabling backup, collaboration, and sharing.

4. .gitignore Keeps Repos Clean

Not everything belongs in version control. Virtual environments, cache files, and secrets should be ignored. This keeps your repository small and secure.

5. The __init__.py File Creates Packages

This special file marks a directory as a Python package, allowing imports like from desktop_agent import main. It can also define package metadata.

Commands Reference

Here are the key commands you learned in this phase:

Virtual Environment

python -m venv venv Create a virtual environment
venv\Scripts\activate Activate it (Windows)
pip install -r requirements.txt Install dependencies

Git Basics

git init Initialize a repository
git add . Stage all changes
git commit -m "message" Create a commit
git status See current state
git log --oneline View commit history

GitHub CLI

gh auth login Authenticate with GitHub
gh repo create name --public Create a new repository
git push -u origin main Push code to GitHub
git push Push subsequent changes
